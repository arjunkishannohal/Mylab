WHAT IS PROTOTYPE POLLUTION?
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
JavaScript objects inherit from Object.prototype. If attacker can modify
this prototype, ALL objects inherit the malicious property.

# Normal object
let obj = {};
obj.isAdmin  // undefined

# After pollution
Object.prototype.isAdmin = true;
obj.isAdmin  // true (inherited!)

WHY IS IT DANGEROUS?
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
1. Security bypass: obj.isAdmin becomes true for ALL objects
2. XSS: Pollute innerHTML/outerHTML sinks
3. RCE (Node.js): Pollute child_process spawn options
4. DoS: Pollute critical application state

COMMON POLLUTION VECTORS:
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
